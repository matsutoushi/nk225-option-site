# 収益化設計(データツールサイト版)

前回サイトの弱点は「PVはあるのにAdSenseのみ」だったこと。今回はページの役割で収益手段を分ける。

## ページ構成と収益の割り当て

| ページ | 役割 | 収益手段 |
|---|---|---|
| トップ(データダッシュボード) | 毎日見に来るリピーターの入口 | AdSense(控えめに)。**成約導線への内部リンクを常設** |
| データ解説ページ(建玉の見方、IVとは等) | 検索流入+初心者の教育 | AdSense+ガイドページへの誘導 |
| /guide/ 口座開設ガイド | 収益の出口 | **アフィリエイト専用(AdSense除外)** |

## 導線の考え方

オプションデータを見に来る人は「すでに取引している人」と「これから始めたい人」の2層。

- 既取引者: 成約は狙わずリピーターとしてPV・拡散に貢献してもらう(AdSense収益)
- 未取引者: 「日経225オプションを取引できる証券会社」ガイドへ誘導(口座開設報酬)
  - 日経225オプション(先物・オプション口座)を扱う主要ネット証券の比較ページを作る
  - 先物・オプション口座の案件があるか各ASP管理画面で確認。なければ総合口座開設案件で代替(総合口座は先物口座開設の前提になるため導線として自然)

## 配置ルール

- データページのファーストビューに広告を置かない(ツールとしての信頼・使い勝手優先。前回のリピーターが戻ってくる理由を守る)
- 全ページ共通ヘッダーに「オプション取引の始め方」リンクを1つだけ常設
- アフィリエイトリンクのあるページには冒頭にPR表記
- X投稿はトップページ(ダッシュボード)へ誘導。「今日のデータ」スクショ+URLの型が基本

## AdSense

- 独自ドメイン設定後に申請(前回運用経験あり)
- 自動広告は /guide/ 配下を除外設定
